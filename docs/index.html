Developer Manual

This manual provides technical documentation for the Sharpishly R&D ecosystem. The application has evolved from a Vanilla JS frontend into a God Mode Agentic Ecosystem supporting autonomous reasoning and secure data management.

Core Principles

  • No Frameworks: Direct DOM manipulation for maximum speed.
  • Single Source of Truth: UI state is derived from app.layout and app.data.
  • Agentic Automation: Background workers for high-stakes tasks.
  • Security First: Master-key encrypted credentials.

Layout System

The application uses a centralized layout object to manage section visibility and titles across the SPA.

layout: {
  1: { title: 'Home', template: 'home' },
  5: { title: 'Settings', template: 'settings-view' },
  6: { title: 'User Profile', template: 'user-profile-view' }
}

By mapping titles to template IDs, the system ensures navigation clicks and rendering remain synchronized.

Agentic Ecosystem Autonomous

Agents reside in php/src/Agents/ and operate as independent loops. They communicate via a file-based job queue.

The Scout Agent

Responsible for competitor analysis and trend scraping. It reports progress via storage/queue/progress.json.

// php/src/Agents/worker.php
while (true) {
    $jobs = glob(STORAGE_PATH . '/queue/*.job');
    foreach ($jobs as $file) {
        $task = json_decode(file_get_contents($file));
        // Process autonomous reasoning...
    }
    sleep(1);
}

Nervous System (SSE)

The bridge (php/nervous_system.php) facilitates real-time communication between the Worker and the UI using Server-Sent Events.

// EventSource Connection in script.js
const sse = new EventSource('/php/nervous_system.php?action=stream');
sse.onmessage = (e) => {
    const data = JSON.parse(e.data);
    updateThoughtStream(data.msg);
};

API Helpers Asynchronous

The frontend abstracts the Fetch API into primary helpers to handle global parsing.

async requestGet(url) {
  const response = await fetch(url);
  return await response.json();
}

async requestPost(url, payload) {
  return await fetch(url, {
    method: 'POST',
    body: JSON.stringify(payload)
  });
}

Secure Vault Encryption

API tokens and sensitive credentials utilize a 256-bit Master Key generated during setup.

public static function encrypt($data) {
    $key = self::getKey(); 
    $iv = openssl_random_pseudo_bytes(16);
    $enc = openssl_encrypt($data, 'aes-256-cbc', $key, 0, $iv);
    return base64_encode($enc . '::' . $iv);
}

Encryption keys are stored outside the public web root in storage/vault/.

PHP MVC & Controllers

Backend routing is handled by a centralized Front Controller pattern located at php/index.php.

1. Controller (e.g., App\Controllers\UserController)
2. Method     (e.g., login)
3. Parameters (e.g., [id])

Every backend request is logged to php/app.log for auditing Milestone #8 API calls.

Docker Orchestration

The environment is split into four primary containers for isolation.

  • sharpishly-app: Nginx Web Server.
  • sharpishly-php: Main MVC API Processor.
  • sharpishly-worker: Background Agent Daemon.
  • sharpishly-db: MySQL 8.0 Persistence Layer.
