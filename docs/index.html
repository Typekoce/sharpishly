Developer Manual

This manual provides technical documentation for the Sharpishly R&D client-side application. The app is built using a "Data-Driven" Vanilla JavaScript architecture designed for high performance and zero-dependency deployments.

Core Principles

  • No Frameworks: Direct DOM manipulation for maximum speed.
  • Single Source of Truth: All UI state is derived from the app.layout and app.data objects.
  • Persistence: Utilization of localStorage for session and project data.

Layout System

The application uses a centralized layout object to manage section visibility and titles.

layout: {
  1: { title: 'Home', template: 'home' },
  5: { title: 'Settings', template: 'settings-view' },
  6: { title: 'User Profile', template: 'user-profile-view' }
}

By mapping titles to template IDs, the system ensures that navigation clicks and page rendering remain perfectly synchronized.

API Helpers Asynchronous

The application abstracts the Fetch API into two primary helpers to handle global error logging and data parsing.

requestGet(url)

async requestGet(url) {
  const response = await fetch(url);
  return await response.json();
}

requestPost(url, payload)

async requestPost(url, payload) {
  return await fetch(url, {
    method: 'POST',
    body: JSON.stringify(payload)
  });
}

Docker Setup

The application is containerized using Nginx to ensure consistent serving across different environments (Windows/Ubuntu).

# docker-compose.yml
services:
  web:
    image: nginx:alpine
    ports:
      - "8080:80"
    volumes:
      - .:/usr/share/nginx/html

To deploy, run: docker-compose up -d
